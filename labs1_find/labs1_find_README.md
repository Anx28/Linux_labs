## Команды, которые отрабатываются

| Команда                              | Что делает                                                                                      | Зачем сисадмину                                                                     |
| ------------------------------------ | ----------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| `find ./workspace -name "*.log"`     | Поиск файлов/директорий по имени или маске. Поддерживает `*` и `?`.                             | Базовый поиск артефактов: логи, конфиги, дампы.                                     |
| `find ./workspace -type f`           | Только **обычные файлы**. Игнорирует директории, симлинки, сокеты.                              | Критично при массовых операциях (`chmod`, `rm`, `tar`), чтобы не сломать структуру. |
| `find ./workspace -empty`            | Пустые файлы или директории.                                                                    | Поиск "мусора" перед очисткой диска.                                                |
| `find ./workspace -type f -mtime +7` | Файлы старше 7 суток.                                                                           | Аудит ротации логов, поиск старых бэкапов.                                          |
| `find ./workspace -type l`           | Только **симлинки**.                                                                            | Проверка целостности деплоя, поиск битых ссылок.                                    |
| `find ./workspace -name "*.txt"`     | Поиск по расширению `.txt`.                                                                     | Переиспользование `-name`.                                                          |
| `find ./workspace ! -name "*.png"`   | Отрицание (`!`). Всё, **кроме** `.png`.                                                         | Инверсивная фильтрация.                                                             |

## Что внутри `workspace/` после generate.sh

```
workspace/
├── logs/
│   ├── app.log
│   ├── nginx.log
│   └── error.log
├── tmp/
│   ├── cache.tmp
│   └── old.tmp           ← состарен на 10 дней через touch -d
├── docs/
│   ├── readme.txt
│   └── notes.txt
├── images/
│   ├── photo.jpg
│   └── logo.png
├── backup/
│   └── archive.tar.gz
├── emptyfile             ← пустой
└── link_to_log → logs/app.log   ← симлинк
```

Структура подобрана так, чтобы каждая задача имела минимум 1–2 матчей и хотя бы один "обманный" файл, который **не должен** попасть в результат.

## Технические детали

- Базовый образ: `ubuntu:24.04`
- WORKDIR: `/app`
- Точка входа: `/bin/bash`
- Запуск с `--rm` — после `exit` контейнер удаляется
- Прогресс не сохраняется между запусками (by design — каждая сессия с чистого листа)
